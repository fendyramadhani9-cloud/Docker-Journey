# 🧪 Docker Journey Hands-on Labs

Selamat datang di **Docker Journey Hands-on Labs**.

Jika Anda berada di halaman ini, berarti Anda telah menyelesaikan seluruh materi pada **Phase 1** dan siap menguji kemampuan melalui serangkaian praktik langsung.

Berbeda dengan Documentation yang berfokus pada konsep dan teori, Hands-on Labs dirancang untuk membantu Anda menerapkan seluruh materi ke dalam skenario yang menyerupai dunia kerja.

Selama Labs berlangsung, Anda akan berperan sebagai **Junior DevOps Engineer** di **PT Venzone Digital**. Setiap Mission akan memberikan tantangan yang berbeda, mulai dari menjalankan container sederhana hingga membangun environment aplikasi menggunakan Docker.

Semakin jauh Anda melangkah, tingkat kesulitan akan meningkat. Seluruh Mission dirancang untuk mempersiapkan Anda menghadapi **Final Lab**, yaitu project akhir yang menggabungkan seluruh materi pada Phase 1.

---

# 🎯 Learning Objectives

Setelah menyelesaikan seluruh Hands-on Labs, Anda diharapkan mampu:

- Menggunakan Docker CLI dengan percaya diri.
- Menerapkan konsep Docker dalam studi kasus nyata.
- Menyelesaikan berbagai tantangan secara mandiri.
- Menggabungkan seluruh materi Docker Fundamentals ke dalam satu project.

---

# 📋 Lab Rules

Sebelum memulai, harap perhatikan beberapa aturan berikut.

- Selesaikan seluruh Documentation sebelum mengerjakan Hands-on Labs.
- Kerjakan setiap Mission secara berurutan.
- Jangan langsung mencari solusi di internet. Cobalah menyelesaikan Mission berdasarkan pemahaman Anda sendiri.
- Jika mengalami kesulitan, pelajari kembali Documentation yang relevan.
- Setiap Mission memiliki **Passing Score minimal 80**.
- Anda dapat mengulang Mission sebanyak yang diperlukan hingga dinyatakan lulus.
- Final Lab hanya dapat dikerjakan setelah seluruh Mission berhasil diselesaikan.

---

# 🤖 Journey CLI

Setelah menyelesaikan sebuah Mission, validasi hasil pekerjaan Anda menggunakan **Journey CLI**.

Contoh:

```bash
journey check docker phase-01 lab-01
```

Journey CLI akan:

- Memvalidasi hasil pekerjaan Anda.
- Memberikan skor akhir.
- Menampilkan requirement yang belum terpenuhi.
- Memberikan rekomendasi Documentation yang perlu dipelajari kembali apabila belum lulus.

Journey CLI **tidak akan memberikan solusi secara langsung**. Tujuannya adalah membantu Anda memahami konsep, bukan menghafal command.
